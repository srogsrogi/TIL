- SSAFY 2학기 첫 프로젝트(6인 개발)

- 전체적인 개발 환경 및 인프라 세팅 담당
- 개발 시작 전부터 이렇게 체계적으로 개발 환경 꽉 잡고 디렉토리 구조, 라이브러리 버전까지 정해놓고 출발하는 건 또 처음이고..
- 아직 초기 개발 중이지만 아직까지 의존성 오류 없이 잘 돌아가는 게 뿌듯해서 정리해 둠



## 파트 별 환경 설정 전략

### 프론트 개발 환경

- conda 가상환경 미사용
- 런타임 버전 고정하고 각 프레임워크의 버전 관리 도구 이용



### 백엔드 개발 환경

- 일반적으로 의존성 문제를 크게 일으키지 않는 python 3.12.9를 고정하고 의존성 맞춰 나감



### AI 개발 환경

- 추론과 전처리에서 서로 다른 의존성(특히 numpy 1 vs 2)을 갖게 될 경우 같은 환경 내에서는 충돌이 일어날 수 있음
- Jetson에서는 어차피 추론쪽 모듈만 필요하니까 전처리와 추론을 분리된 환경에서 실행하는 것도 가능하나, 개발 편의를 위해 하나의 환경으로 통일
- 최신 버전을 사용하더라도 추론 속도는 크게 차이나지 않을 것으로 예상
- 전처리 속도는 10% 내외 차이날 것으로 예상
- 속도보다 안정성을 우선하자는 의사결정에 따라 numpy 1.26.4를 고정하고 나머지 라이브러리의 의존성을 맞춰나감



### MLOps 개발 환경

- 독립된 플랫폼이기 때문에 큰 의존성 문제가 발생할 가능성이 낮아 최신성을 유지
- 현시점 dockerhub 배포된 이미지 기준 latest 버전 사용
